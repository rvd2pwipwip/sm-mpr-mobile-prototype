# SM Search & Browse – UX Principles

## Purpose

Search & Browse provide complementary pathways to explore the full breadth of the catalog. While Browse supports structured discovery, Search enables direct access through user intent, together empowering deep exploration and efficient retrieval.

---

## Core UX Principles

### 1. Dual Discovery Model: Explore vs. Retrieve

- Browse enables hierarchical exploration of the catalog.
- Search supports direct, query-driven retrieval of specific content.
- Both coexist within a unified experience and dynamically take precedence based on user intent.

**UX intent:** Support both exploratory and goal-oriented behaviors without forcing users into one mode.

---

### 2. Progressive Focus Shift

- The interface begins in Browse mode, with Search visually lightweight.
- As soon as the user engages with Search (typing), the experience transitions to fully prioritize results.
- Browse gracefully recedes to give Search full attention.

**UX intent:** Reflect user intent through adaptive interface priority.

---

### 3. Catalog-Scale Navigation

- Browse organizes large content libraries (music, podcasts, radio) through multi-level hierarchies.
- Structure adapts based on catalog size and regional availability.

**UX intent:** Make large-scale content navigable regardless of catalog complexity or territory.

---

### 4. Adaptive Information Architecture

- Browse dynamically adjusts structure:
  - Smaller catalogs: simplified (e.g., genres only)
  - Larger catalogs: deeper hierarchies (Genre, Mood, Activity, Era, Theme)
- Conditional categories appear only when relevant (e.g., personalized podcast sections).

**UX intent:** Balance depth and simplicity based on context and user value.

---

### 5. Contextual Personalization

- Both Browse and Search incorporate user-specific data:
  - Personalized podcast categories (Continue Listening, Downloads, etc.)
  - Relevant radio stations by location and popularity
- Presentation adapts to territory and usage.

**UX intent:** Increase relevance by aligning discovery with user context and behavior.

---

### 6. Incremental Search Feedback

- Search activates immediately as users type.
- Results update dynamically with each character entered.
- Interface elements adapt (e.g., search icon becomes clear action).

**UX intent:** Provide responsive, real-time feedback to support fast refinement.

---

### 7. Structured Result Organization

- Search results are grouped into meaningful categories:
  - Channels, Artists, Podcasts, Episodes, Radio
- Only populated categories are displayed.
- Each category provides access to deeper result sets via “More.”

**UX intent:** Help users quickly scan and navigate across different content types.

---

### 8. Seamless Mode Switching

- Users can easily switch between Search and Browse:
  - Clearing the query restores Browse.
  - Navigation reinforces the reset to the default state.
- No loss of context or friction when moving between modes.

**UX intent:** Keep transitions fluid and reversible to support exploration.

---

### 9. Familiar Interaction Patterns

- Uses consistent UI patterns shared across the app:
  - Swimlanes and “More” views
  - Grid layouts for expanded content
  - Persistent header and footer chrome

**UX intent:** Leverage consistency to reduce learning curve and increase efficiency.

---

### 10. Content-Type-Aware Discovery

- Each content type follows tailored discovery logic:
  - Music: hierarchical taxonomy (genre, mood, etc.)
  - Podcasts: category-rich exploration and user-driven sections
  - Radio: geolocation and format-based navigation

**UX intent:** Respect the unique nature of each content type for optimal discoverability.

---

### 11. Geo-Relevant Filtering and Discovery

- Radio and music adapt to user location and market availability.
- Local relevance is prioritized while still enabling global exploration.

**UX intent:** Balance familiarity (local relevance) with discovery (global access).

---

### 12. Layered Interface with Persistent Chrome

- Header (search + tabs) and footer remain fixed.
- Content scrolls beneath, with proper spacing and accessibility.

**UX intent:** Maintain orientation and control while maximizing content visibility.

---

## In Summary

Search & Browse form the app’s deep discovery engine: a flexible, scalable system that adapts to user intent, catalog size, and personalization signals. By combining structured exploration with real-time search, the experience supports everything from casual discovery to power-user precision, all within a cohesive and responsive interface.
